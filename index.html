「Webコンテンツ作成」の成果

本テーマについて

先行する情報工学の専門科目の基礎知識を応用して実践的な能力を修得することが目的です。その一つとしてHTML等を用いたWebページ作成が挙げられます。なお課題に対する解決能力育成にも寄与します。

プロジェクト課題について

世界の天気

私はプロジェクト課題で天気の検索を行えるページを作成しました。天気を選んだ理由は天気によって背景に変化が現れたら面白いなと思ったからです。少し時間がかかりましたが、実際に天気が小雨のシドニーを検索すると背景に雨のアニメーションを追加する仕組みを実装することができました。もう一つ注力したポイントがあります。それは検索ページに地図を表示することです。Google Mapsを利用してページに簡単に地図を挿入することができました。この機能自体はAPIではなく、Google Mapsにもともと備わっている機能だったためペーニ表示すること自体はとて簡単でした。

プロジェクト課題で作成したページ: project.html

数当てゲーム

判定処理を記述しました。グローバル変数から何度目の判定であるかを取得し、試行回数と予測した数値と実際の答えの数値を引数に処理を行い、適切な返答を出力するようif文を主に使ったプログラムを行いました。

コンソールでのみ表示されていたゲームのリザルトをJavaScriptを用いてHTMLを操作し、実際に画面に表示されるようにページを変更しました。具体的にはtextContentを持ちて指定されたIDのspan要素の中身を編集しています。

ページ内にボタンとインプット要素を配置し、実際にゲームが行えるように編集しました。今までは事前に設定されていたユーザーが入力する値を今回からはインプットに数値を記述し、回答ボタンを押すことで判定メソッドに渡され正解か不正解かを判定できるようになりました。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第１回 WWW と HTML

HTMLの基本的な記述、Gitの使い方、HTML要素の調べ方などWebページを作る上で必要になってくる基礎的な技術を学びました。またHTMLのデバッグを行うサイトなど作成する上で役立つ知識も身につけました。

演習の成果: 演習1-4:HTMlのさまざまな要素

第２回 CSS による web ページの作成

CSSを用いてwebページの装飾を行いました。CSSの記述方法やHTMLとどのように関連付けるかなど基礎を主に学びました。また思うようにwebページに反映されなかった時のデバック方法、その流用を学びより簡単に様々な装飾を試せるようになりました。

演習の成果: 練習問題2-3:ウェブブラウザの紹介

第３回 JavaScript の 基本

JavaScriptについて学びました。JavaScriptと名前の似ているjavaとの相違点やプログラムを書く上での注意点、長所とも短所とも言える特徴も学びました。特に大切だと感じた内容はデバックの方法です。エラーの出た箇所でプログラムを停止させる方法や、ブレイクポイント使った意図しない動きへの対処なども学びました。

演習の成果: 練習3-6:オブジェクトの参照

第４回 DOM と JavaScript

JavaScriptを用いてHTML・CSSを操作する方法を学びました。これによってボタンなどによってアクションが発生した際にページ内を変更することができるようになりました。具体的には要素の作成、要素の内のテキストの編集、要素の削除です。

演習の成果: 練習問題4-2~5

第５回 イベント処理

ボタンやインプットなどアクション関連の事を学びました。これによってボタンをクリックすることで何かしらのメソッドを呼び出すことができるようになりました。またインプット要素内の値を取得し判定に使用したり、出力に利用できるようになりました。

演習の成果: 練習問題5-3

工夫点、苦心点、強調点、感想

webページを作るにあたって、デザインのことはあまりわからないのでシンプルかつ単調すぎないようなデザインになることを意識して作りました。色選びについても温かみがある見やすい色を選びました。数当てゲームを作る際にはJavascriptからHTMLの文章を編集することにすこし苦労しました。内部の処理についてはプログラムが少し得意だったのであまり苦労せずに完成させることができました。プロジェクト課題では世界の天気を選び制作を進めていましたが、HTMLをJavascriptで書く課題で少し躓きました。数当てゲームで少し苦労した分、書き方はなんとなくわかってはいたのですが量が多くなってくると、要素同士の配置で混乱してしまい、うまく配置できずに大変な思いをしました。ただ、ほぼ全ての要素をJavascriptで作り、画面に表示しているのでその後の表示内容を変更したり、イベントを設定したりする際にはかなり楽をすることができました。そして私のプロジェクト課題には注目して欲しいポイントが二つあります。一つは画面左側に大きく表示されている地図です。この地図はAPIではなくGoogle mapsの機能でURLをiframe要素で使うことで表示しています。URLの一部を検索キーでヒットしたデータから取り出した経度と緯度に書き換えることで検索場所を地図に表示することもできるようになりました。初めは経度と緯度をURLに記載してもなかなか思ったように地図が変化せず苦労しました。今のところは経度と緯度で表示す場所を変更するのみですがGoogle Mapsには他にも機能があり、例えばズームアウトした状態からなどの機能があります。本当ならズームアウトした状態で表示したいのですがどうしてもうまく機能してくれないので断念しました。URLの埋め込みではなくAPIとして使用すればこういったことは起らなくなうようですがAPIは少し難しかったので諦めました。もう一つの注目してほしいポイントはアニメーションです。実は検索結果の天候が雨になるとbackgroundに雨のアニメーションが流れるような作りになっています。アニメーションとはいってもただGIF画像をimg要素として表示しているだけなのでとても簡単です。しかし表示すると他の要素の上に表示されてしまったり、逆に背景色の後ろに表示されたりしてなかなか思うように表示することが出来ませんでした。しかしz-indexを調節することで表示したい順に要素を表示できるようになりました。今回のwebサイトの作成ではわからないときの調べかたや、デバックの方法まで一通り教わったのでwebサイトを作る能力がそれなりに身についたと思います。将来webページ系の仕事に就くことがあればこの能力を活かして仕事をしていきたいです。
